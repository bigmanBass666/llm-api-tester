# Checklist

## 阶段一：URL 与分页逻辑验证
- [ ] URL 已正确添加 pageSize=96 参数
  - 验证方法：检查 scraper.py 中 URL 包含 `&pageSize=96` 或 `?pageSize=96`
  - 验证两种排序均已添加（popular 和 recent）
- [ ] 新的分页方法 `_go_to_next_page()` 已实现
  - 验证方法：方法存在且包含以下逻辑：
    - 定位下一页按钮: `button[aria-label="Go to next page"]`
    - 点击按钮并等待 2-3 秒
    - 返回 bool 表示是否成功
    - 异常处理（disabled、不存在等）
- [ ] 主循环已重构为分页遍历模式
  - 验证方法：`scrape_models()` 的 while 循环条件不再依赖滚动
  - 循环体内调用 `_go_to_next_page()` 而非 `_scroll_for_more()`
  - 有最大翻页次数限制（≤10）
  - 有智能终止条件（next button disabled 或连续无新模型）

## 阶段二：选择器与提取验证
- [ ] 模型卡片选择器已更新为精确版本
  - 验证方法：使用 `[data-testid="nv-card-root"]` 替代旧的宽泛选择器
  - 选择器在测试中能准确定位所有模型卡片
- [ ] 数据提取逻辑已增强
  - 验证方法：ModelInfo 对象包含以下新增/优化字段：
    - vendor (str): 发布商名称 ✅
    - tags (list): 标签列表 ✅
    - is_downloadable (bool): 基于 tags 判断 ✅
    - is_free_endpoint (bool): 基于 tags 判断 ✅
    - id (str): 完整格式 `{vendor}/{model}` ✅
- [ ] 废弃代码已清理
  - 验证方法：旧的 `_scroll_for_more()` 已注释或删除
  - 旧的 `_fallback_extract()` 已注释或删除
  - 代码中无残留的无用滚动逻辑

## 阶段三：功能测试验证
- [ ] 小规模测试通过（10 个模型）
  - 验证方法：运行 `-n 10 --scrape-only` 成功返回 ≥10 个不重复模型
  - 无报错或异常终止
  - 日志输出正常（控制台简洁，文件详细）
- [ ] 中等规模测试通过（100 个模型）
  - 验证方法：运行 `-n 100 --scrape-only` 成功返回 ≥90 个模型
  - 总耗时 < 60 秒（合理范围）
  - 翻页逻辑正常工作（能看到多页遍历的日志）
- [ ] 全量测试通过（200 个模型，可选）
  - 验证方法：运行 `-n 200 --scrape-only` 成功返回 ≥150 个模型（接近 191）
  - 覆盖率显著提升（对比优化前的 24 个）
  - 报告显示完整的模型列表和统计信息

## 整体验收标准
- [ ] 核心目标达成：能获取远超 24 个模型的数量（目标 ≥150）
- [ ] 分页机制正常：自动遍历多页直到获取足够模型或到达末页
- [ ] 选择器精确：使用 data-testid 选择器，无误匹配
- [ ] 数据完整性：模型对象包含 vendor、tags 等完整字段
- [ ] 代码质量：旧代码已清理，新代码有注释，无冗余逻辑
- [ ] 向后兼容：如有必要可通过参数调整 pageSize（默认 96）
- [ ] 性能合理：全量爬取耗时 < 2 分钟（含网络延迟）
