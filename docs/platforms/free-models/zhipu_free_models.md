## 一、文本对话模型

### 1. GLM-4.7-Flash（主力推荐）

| 属性 | 值 |
|------|----|
| 模型 ID | `glm-4.7-flash` |
| 架构 | 30B-A3B (MoE) |
| 上下文窗口 | **200K** tokens |
| 最大输出 | 128K tokens |
| 价格 | **免费** |
| 并发限制 | 1（V0 等级，可随权益提升）|
| 发布日期 | 2026-01-19 |

**能力支持**: 思考模式、流式输出、Function Call、上下文缓存、结构化输出、MCP 工具调用

**核心特性**:
- 面向 Agentic Coding 场景优化，SWE-bench Verified 59.2 分
- 支持 tool use / function calling
- 支持 `stream=True` 流式输出
- 开源模型，Hugging Face 可下载，vLLM / Ollama 原生支持

**并发 = 1 的实际影响**:
- 单线程串行调用完全够用
- 1 小时连续单线程约 120 次请求 / 18 万 token 输出
- 平均首 token 延迟约 0.8s，输出速率约 65 tokens/s
- 不适合多用户在线服务、高并发生产环境

**适用场景**: 个人项目、原型验证、异步批处理、编辑器插件、单用户 Agent 任务

**不适用场景**: 多用户服务、实时聊天产品、长链 Agent 工作流

## 五、速率限制机制

### 查看方式

登录 [官方速率限制控制台](https://www.bigmodel.cn/usercenter/proj-mgmt/rate-limits) 查看当前账户各模型的并发数。

> **重要**: 并发数是**按账户+按模型独立计算**的，不是全局共享。当前账户处于 V0 等级。

### 限制维度

智谱的速率限制基于**并发请求数**（非 RPM/TPM），即同一时刻正在处理中的请求数量。

- **每个免费模型有独立的并发上限**
- **不同用户权益等级对应不同的并发限制**
- **高峰期（工作日 15:00-18:00）可能触发动态限流**

### 错误码

| 错误码 | 含义 | 处理方式 |
|--------|------|---------|
| 1302 | 触发用户速率限制 | 降低并发、增加排队机制 |
| 1305 | 平台服务过载 | 稍后重试、增加重试间隔 |

### 提升方式

- 通过 [速率限制调整申请](https://bigmodel.cn/rate-limits/form) 提交申请
- 提升用户权益等级（通过积分）
- 审核周期约 10 个工作日

## 九、接入示例

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["ZHIPU_API_KEY"],
    base_url="https://open.bigmodel.cn/api/paas/v4"
)

# 文本对话
response = client.chat.completions.create(
    model="glm-4.7-flash",
    messages=[{"role": "user", "content": "你好"}],
    max_tokens=256,
    stream=True
)
for chunk in response:
    print(chunk.choices[0].delta.content or "", end="", flush=True)

# 图像理解
response = client.chat.completions.create(
    model="glm-4v-flash",
    messages=[{
        "role": "user",
        "content": [
            {"type": "text", "text": "描述这张图片"},
            {"type": "image_url", "image_url": {"url": "https://example.com/image.png"}}
        ]
    }]
)
```
